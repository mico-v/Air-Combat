# 飞机大战 - 零基础入门教程

## 项目简介

这是一个基于 Windows Win32 API 的飞机大战游戏框架，专为零基础的初学者设计。本项目提供完整的代码框架、详细的注释和分步骤的文档，帮助你从零开始学习游戏开发。

## 开发环境要求

### 必需软件
1. **操作系统**: Windows 10 或更高版本
2. **开发工具**: Visual Studio 2019 或更高版本
   - 下载地址: https://visualstudio.microsoft.com/zh-hans/downloads/
   - 安装时需要选择 "使用 C++ 的桌面开发" 工作负载

### 可选工具
- Git (用于版本控制)
- 图片编辑工具 (如 Paint.NET, GIMP 等，用于制作游戏资源)

## 快速开始

### 第一步：克隆或下载项目

```bash
# 使用 Git 克隆
git clone https://github.com/mico-v/Air-Combat.git

# 或直接下载 ZIP 文件并解压
```

### 第二步：打开项目

1. 找到项目文件夹中的 `Game.sln` 文件
2. 双击打开 (会自动启动 Visual Studio)
3. 如果提示升级项目，选择"确定"

### 第三步：编译运行

1. 按键盘 `F5` 键或点击菜单栏的 "调试" -> "开始调试"
2. 第一次编译可能需要几分钟时间
3. 编译成功后会自动运行游戏

## 学习路径

### 零基础学习路线

如果你是完全的零基础，建议按以下顺序学习：

#### 第一阶段：准备工作 (1-2天)
1. 阅读 `SETUP.md` - 了解如何搭建开发环境
2. 阅读 `docs/01-环境搭建.md` - 详细的环境配置步骤
3. 成功编译并运行项目

#### 第二阶段：理解框架 (3-5天)
1. 阅读 `docs/02-框架概览.md` - 了解整体架构
2. 阅读 `docs/03-Win32基础.md` - 学习 Win32 基础知识
3. 阅读 `docs/04-游戏核心.md` - 理解游戏循环原理
4. 阅读代码中的注释，理解每个函数的作用

#### 第三阶段：实践开发 (1-2周)
1. 阅读 `docs/05-场景系统.md` - 学习如何创建场景
2. 阅读 `docs/06-游戏对象.md` - 学习如何添加游戏对象
3. 阅读 `docs/07-UI系统.md` - 学习如何制作UI
4. 跟随 `GuideBook.md` 中的教程一步步实现功能

#### 第四阶段：进阶开发 (1-2周)
1. 阅读 `docs/08-进阶功能.md` - 学习更多高级功能
2. 自己添加新功能，如：
   - 新的敌人类型
   - 道具系统
   - 关卡系统
   - 音效和背景音乐

## 核心概念

### 1. 事件驱动编程
Windows 程序通过响应事件（如鼠标点击、键盘按下）来执行操作。

### 2. 游戏循环
游戏以固定频率（如每秒60次）更新游戏逻辑和绘制画面。

### 3. 场景管理
游戏分为不同的场景（开始界面、游戏场景、结束界面等），每个场景有自己的逻辑。

### 4. 对象管理
游戏中的角色、敌人、子弹等都是游戏对象，需要统一管理它们的创建、更新和销毁。

## 常见问题 (FAQ)

### Q1: 编译时出现错误怎么办？
A: 
1. 检查是否安装了正确的 Visual Studio 工作负载
2. 查看错误信息，通常会提示缺少什么
3. 参考 `docs/常见问题.md` 文档

### Q2: 如何添加新的图片资源？
A:
1. 准备 BMP 格式的图片
2. 在 Visual Studio 中打开资源视图
3. 添加位图资源并设置 ID
4. 在代码中加载和使用

详细步骤见 `docs/06-游戏对象.md`

### Q3: 如何调试程序？
A:
1. 按 F9 在代码行上设置断点
2. 按 F5 开始调试
3. 程序会在断点处暂停
4. 可以查看变量值、单步执行等

详细说明见 `docs/调试技巧.md`

### Q4: 游戏运行很卡怎么办？
A:
1. 检查是否有内存泄漏（对象创建后没有销毁）
2. 优化碰撞检测算法
3. 减少不必要的绘制操作

## 开发技巧

### Visual Studio 常用快捷键

- `F5` - 开始调试
- `Ctrl + F5` - 运行（不调试）
- `F9` - 设置/取消断点
- `F10` - 单步跳过
- `F11` - 单步进入
- `Ctrl + K, Ctrl + C` - 注释选中代码
- `Ctrl + K, Ctrl + U` - 取消注释
- `F12` - 转到定义
- `Ctrl + -` - 返回上一个位置

### 代码规范

本项目遵循以下代码规范：
1. 变量名使用小驼峰命名法 (例如: `playerSpeed`)
2. 函数名使用大驼峰命名法 (例如: `CreatePlayer`)
3. 常量使用全大写加下划线 (例如: `WINDOW_WIDTH`)
4. 每个函数都有详细的中文注释说明
5. 复杂逻辑有行内注释解释
